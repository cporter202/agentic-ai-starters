# Architecture

## Core components

- competitor registry and watch rules
- page snapshot collector
- signal collectors for traffic, tech stack, and hiring cues
- change-detection layer
- synthesis agent for narrative-level insights
- scheduled briefing output

## Suggested flow

1. Capture baseline snapshots for each monitored competitor.
2. Revisit watch surfaces on a defined cadence.
3. Detect messaging, pricing, and structure changes.
4. Add supporting signals like traffic direction or stack changes.
5. Produce an operator brief with implications and follow-up ideas.

## MVP notes

- Start with five to ten competitors, not fifty.
- Separate raw deltas from strategic interpretation so analysts can audit the output.
- Save HTML or markdown snapshots for historical comparisons.
