# Architecture

## Core components

- keyword and topic intake
- SERP research collector
- competitor page extractor
- brief generator
- drafting agent
- editorial QA and factuality pass
- publishing handoff to Notion or CMS

## Suggested flow

1. Normalize the keyword target and desired business outcome.
2. Pull SERP snapshots and competing page structures.
3. Create a brief with intent, outline, FAQs, and link targets.
4. Draft the article in the requested voice and format.
5. Run checks for thin sections, unsupported claims, and duplicate intent.

## MVP notes

- Keep briefing and drafting as separate steps so editors can approve structure first.
- Save SERP and competitor snapshots with timestamps because search landscapes change.
- Add a human review gate before publication for factual or medical claims.
